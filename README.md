## 🚀 Key Projects

### 1. Azure 기반 멀티 에이전트 주식 시장 시뮬레이션 및 데이터 엔지니어링 파이프라인 (`2026.06 ~ 진행 중`)
* **설명:** 사용자의 주문이 시장 가격 체결 구조 및 에이전트 행동에 미치는 영향을 관찰할 수 있는 가상 주식 시뮬레이션 환경 구축 및 실시간 트랜잭션 분석 파이프라인 프로젝트[cite: 3].
* **역할:** **팀장 및 데이터 엔지니어 / 아키텍처 설계 총괄**[cite: 2, 3]
* **사용 기술:** `Azure Event Hubs`, `Stream Analytics`, `Azure Databricks`, `Azure SQL Database`, `Azure OpenAI`, `Python`[cite: 2, 3]
* **상세 수행 내용:**
  * **아키텍처 및 저장소 분리:** 시뮬레이션의 목적과 트랜잭션 특성에 맞춰 실시간 데이터 처리 레이어와 장기 배치 분석 레이어를 분리 배치하고 전체 클라우드 시스템 인프라 아키텍처를 설계[cite: 2, 3].
  * **스트리밍 파이프라인:** 체결 엔진에서 실시간으로 발생하는 대규모 거래 및 에이전트 행동 로그 데이터를 `Azure Event Hubs`를 통해 인제스션(Ingestion)하고, `Stream Analytics`로 실시간 가공 처리 흐름 구현[cite: 3].
  * **분석용 데이터 파이프라인:** `Azure Databricks`를 활용해 사용자별 거래 패턴 및 에이전트 분류 행동 유형 데이터를 적재·전처리하고, `Azure SQL Database`로 연동하여 어드민 대시보드(Admin Dashboard) 환경 지원[cite: 3].
  * **AI 변수 연동:** `Azure OpenAI`를 연동하여 뉴스, 금리 등 외부 변수에 따른 에이전트의 투자 판단 심리 지표 데이터가 파이프라인 및 테이블 스키마 구조 내에 자연스럽게 녹아들도록 흐름 정립[cite: 3].
* **기술적 트러블슈팅 (Troubleshooting):**
  * **[문제점]** 시뮬레이션 규모에 따른 동시다발적 체결 이벤트 발생 시 데이터 적재 지연 및 정형 DB 저장소 과부하 위험성 발견[cite: 3].
  * **[해결책]** 데이터 성격과 처리 목적(실시간 통계 vs 장기 패턴 분석)에 따른 전용 데이터 인프라(Databricks 레이어 및 스토리지)를 분리하고, 대시보드 조회 조건에 맞춰 테이블 인덱싱 구조 및 쿼리 캐싱 파이프라인을 최적화하여 데이터 레이텐시 최소화[cite: 2, 3].

### 2. Azure 기반 실시간 EV 배터리 이상 탐지 모니터링 시스템 (`2026.03 ~ 2026.05`)
* **설명:** 실시간 차량 센서 데이터를 클라우드로 수집·처리하고, 이상 탐지 결과를 SQL 인프라와 대시보드로 연동한 End-to-End 파이프라인 프로젝트[cite: 2]
* **역할:** **팀장 및 데이터 엔지니어** (전체 아키텍처 구조 설계, 데이터 흐름 구현)[cite: 2]
* **사용 기술:** `Microsoft Azure`, `Azure IoT Hub`, `Stream Analytics`, `Azure SQL DB`, `Azure Functions`, `Python`[cite: 2]
* **상세 수행 내용:**
  * **Lambda Architecture 적용:** 실시간 스트리밍 흐름과 배치성 분석 흐름의 역할을 효율적으로 이원화하여 대규모 인프라 부하 분산 설계[cite: 2].
  * **스트리밍 수집 가공:** Python 기반 시뮬레이터로 생성된 차량 센서 데이터를 Azure IoT Hub로 유실 없이 수집하고, Stream Analytics를 통해 실시간 이상 데이터 처리 및 필터링 로직 구현[cite: 2].
  * **데이터 저장 및 서빙:** 처리 결과를 수집 시점부터 서비스 활용 시점까지 고려하여 Azure SQL Database 스키마 구조에 최적화 적재하고, Azure Functions API 기반으로 웹 대시보드 화면에 데이터 지연 없이 연동 검증[cite: 2].

### 3. 딥러닝 기반 딸기 숙성도 분류 및 로봇 팔을 활용한 수확 자동화 시스템 (`2025.01 ~ 2025.10`)
* **소속:** 국립한밭대학교 임베디드 연구실 (DfX) - 졸업 캡스톤 프로젝트[cite: 2]
* **설명:** 카메라 비전 기술로 딸기의 성숙도와 위치를 딥러닝으로 인식하여, 물리적 로봇팔로 타겟팅 자동 수확을 제어하는 인공지능 기반 임베디드 통합 시스템[cite: 2]
* **역할:** **임베디드 제어 코드 개발 및 비전-제어 좌표계 연동 흐름 구현**[cite: 2]
* **사용 기술:** `Python`, `Nvidia Jetson Nano`, `Embedded Robot Control`, `Deep Learning Object Detection`[cite: 2]
* **상세 수행 내용:**
  * **통신 아키텍처 구현:** 에지 디바이스인 `Jetson 환경`과 물리 로봇팔 하드웨어 장치 간의 소켓/시리얼 통신 프로토콜을 정립하여 데이터를 안정적으로 송수신하는 파이프라인 구축[cite: 2].
  * **좌표계 변환 알고리즘 적용:** 카메라 비전 센서로 탐지된 2D/3D 이미지 상의 딸기 위치 픽셀 좌표 정보를 로봇팔이 인식하고 동작할 수 있는 실제 3차원 공간 좌표계로 변환하는 정밀 연동 제어 로직을 Python 코드로 구현[cite: 2].
  * **동작 검증 프로세스:** 객체 인식 모델의 딸기 숙성도 분류 알고리즘 결과값에 따라 '수확/미수확' 의사결정을 트레이싱하고, 로봇팔 통신 및 이동 제어 테스트 시나리오를 반복 설계하여 전체 제어 시스템의 안정성 검증[cite: 2].
